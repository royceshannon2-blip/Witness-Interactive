# Version 1.4.2 Release Summary

## What Was Fixed

### 1. ✅ Knowledge Checkpoint Quiz Now Displays
**Problem**: Quiz wasn't showing after completing missions due to incorrect PathClassifier method call.

**Solution**: 
- Fixed `KnowledgeCheckpoint.js` to use `PathClassifier.classify()` as a static method
- Fixed `UIController.js` to pass `PATH_RULES` instead of `roleId` to the classifier
- Quiz now displays correctly with 5 path-aware questions

**Files Changed**:
- `js/engine/KnowledgeCheckpoint.js`
- `js/engine/UIController.js`

### 2. ✅ Psychology Score Notifications Added
**Problem**: Players couldn't see how their choices affected their psychology scores in real-time.

**Solution**:
- Added toast notifications that appear when scores change
- Shows score name, delta (+/- value), and direction arrow
- Notifications slide in from the right, stagger for multiple changes, and fade out after 2.5 seconds
- Respects `prefers-reduced-motion` accessibility setting
- Mobile-responsive design

**Files Changed**:
- `js/engine/MoraleHUD.js` - Added `showScoreChangeToasts()` method
- `css/style.css` - Added toast notification styles

**Features**:
- Green notifications for positive changes (↑ +5)
- Red notifications for negative changes (↓ -3)
- Staggered animations when multiple scores change
- Accessible with ARIA live regions

### 3. ✅ Google Sheets Integration Documentation
**Problem**: No way to collect and view feedback survey responses.

**Solution**:
- Created comprehensive setup guide at `docs/google-sheets-integration.md`
- Includes step-by-step instructions for:
  - Creating a Google Sheet
  - Setting up Google Apps Script webhook
  - Deploying as a web app
  - Configuring the feedback system
  - Testing the integration

**What You Need to Do**:
1. Follow the guide in `docs/google-sheets-integration.md`
2. Create your Google Sheet and Apps Script
3. Update `config/feedback-config.js` with your webhook URL
4. Change `mode` from `'console'` to `'http'`
5. Commit and push the changes

**Data Collected**:
- Timestamp
- Session ID (anonymous)
- Mission and Role IDs
- Rating (1-5 emoji scale)
- What players liked (optional text)
- What felt broken (optional text)
- Survival status and probability
- All choice flags (JSON)
