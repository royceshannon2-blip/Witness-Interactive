🔍 Rwanda Mission - Accessibility Testing

📋 Testing Instructions

This page helps you manually test the Rwanda Genocide Mission for WCAG AA accessibility compliance.

Requirements: NFR-3.1 - All content must be screen-reader compatible, keyboard navigable, mobile responsive, and meet WCAG AA color contrast standards.

Test 1: Screen Reader Compatibility (NVDA/JAWS)

Setup:

  1. Install NVDA (free) from nvaccess.org
  2. Start NVDA (Ctrl+Alt+N)
  3. Open the game in a new tab: http://localhost:8000

Hutu Moderate Role

  • Screen reader announces "Rwanda, 1994" button
  • "Hutu Moderate" role button is announced with description
  • Narrative text is announced when scene loads
  • Choice buttons have clear accessible names
  • Timer countdown is announced (if present)

Tutsi Survivor Role

  • "Tutsi Survivor" role is announced
  • Timed choice timer is announced via ARIA live region
  • Scene transitions are announced

UN Peacekeeper Role

  • "UN Peacekeeper" role is announced
  • All narrative text is accessible
  • Outcome screen is accessible
  • Historical ripple timeline is accessible

Test 2: Keyboard Navigation

Test with keyboard only (no mouse):

  1. Use Tab to move forward through interactive elements
  2. Use Shift+Tab to move backward
  3. Use Enter or Space to activate buttons
  4. Use Escape to close modals (if any)

Tab Navigation

  • Tab key moves focus through all interactive elements
  • Focus order is logical (top to bottom, left to right)
  • Focus indicator is visible on all elements
  • No keyboard traps (can always move focus away)

Enter/Space Activation

  • Enter key activates focused buttons
  • Space key activates focused buttons
  • Both keys work consistently

Full Keyboard Playthrough

  • Can start game with keyboard only
  • Can select Rwanda mission with keyboard
  • Can select role with keyboard
  • Can play through 3-5 scenes with keyboard only
  • Can complete mission using only keyboard

Test 3: Mobile Viewport Testing

Test on different viewport sizes:

Open DevTools (F12), click device toolbar icon, and test these viewports:

320px Viewport (Small Mobile)

  • Mission button is visible and tappable
  • Role selection buttons fit on screen
  • No horizontal scrolling
  • Narrative text is readable
  • Choice buttons are tappable (min 44×44px)
  • Timer display is visible
  • No content is cut off

768px Viewport (Tablet)

  • Layout adapts to tablet size
  • Content is centered or well-distributed
  • Text is readable without zooming
  • Buttons are appropriately sized
  • No awkward spacing or gaps

1280px Viewport (Desktop)

  • Content is centered
  • Max-width constraints are applied
  • Text is readable
  • Layout is balanced
  • No excessive whitespace

Test 4: Color Contrast (WCAG AA)

WCAG AA Requirements:

  • Normal text (16px): 4.5:1 contrast ratio
  • Large text (18px+): 3:1 contrast ratio
  • UI components: 3:1 contrast ratio

Tools:

  • Chrome DevTools Lighthouse (F12 → Lighthouse tab)
  • WebAIM Contrast Checker
  • Browser extension: WAVE or axe DevTools

Contrast Samples

Narrative Text

#e0e0e0 on #1a1a1a

Choice Button

#ffffff on #2196F3

Timer Display

#ff9800 on #333

Focus Indicator

#4CAF50 on #2a2a2a

Contrast Checklist

  • Narrative text: 4.5:1 or higher
  • Choice button text: 4.5:1 or higher
  • Choice button border: 3:1 or higher
  • Focus indicator: 3:1 or higher
  • Timer display: 4.5:1 or higher
  • Text remains readable during atmospheric effects
  • Opacity doesn't drop below 0.7 during effects

Test 5: Focus Management

Focus Visibility

  • Focus indicator is always visible
  • Focus indicator has sufficient contrast
  • Focus indicator is not hidden by other elements

Focus Order

  • Focus moves in logical order
  • Focus doesn't jump unexpectedly
  • Focus order matches visual order

Focus During Transitions

  • Focus is not lost during scene transitions
  • Focus moves to meaningful element after transition
  • Focus is not on body element

Test 6: Reduced Motion Support

Enable Reduced Motion:

  • Windows: Settings → Ease of Access → Display → Show animations (OFF)
  • Mac: System Preferences → Accessibility → Display → Reduce motion (ON)
  • Browser: DevTools (F12) → Rendering → Emulate CSS media feature prefers-reduced-motion: reduce

Reduced Motion Checklist

  • Typewriter animation is disabled (text appears instantly)
  • Scene transitions are instant or very fast
  • Atmospheric effects are reduced or disabled
  • Timer pulse effects are disabled
  • Choice button hover effects are reduced
  • Game still works correctly
  • No features are broken
